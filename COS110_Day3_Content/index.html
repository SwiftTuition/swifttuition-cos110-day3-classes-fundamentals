Day 3: Introduction to Classes & Object-Oriented Programming

Transform from procedural thinking to object-oriented mastery - learn to create intelligent, self-contained code structures

Welcome to Day 3!

Welcome to Day 3 of your COS 110 crash course! Today we're making a fundamental shift in how we think about programming. We're moving from procedural programming (where we organize our code around functions) to object-oriented programming (where we organize our code around classes and objects).

Think of it like this: Instead of having a toolbox full of separate tools (functions) that you have to remember how to use correctly, we're going to learn how to create smart, self-contained machines (objects) that know how to do their own work and protect themselves from being used incorrectly!

Real-world Analogy

Old way (Procedural): Having loose parts of a car engine scattered around your garage, and you have to remember which bolts go where

New way (Object-Oriented): Having a complete, working car where you just turn the key and drive - all the complex parts work together automatically

Learning Objectives

By the end of this day, you will be able to:

  1. Explain why classes are useful and understand the limitations of procedural programming
  2. Define a class using proper C++ syntax with member variables and member functions
  3. Use access specifiers (public and private) to control access to class members
  4. Create objects (instances) from a class and use the dot operator to access members
  5. Write and use constructors to initialize objects when they are created
  6. Understand the scope resolution operator (::) and how to implement member functions

1. Why Classes? The Problem with Procedural Programming ü§î

The Traditional Approach: Procedural Programming

What is Procedural Programming?

Procedural programming is like having a workshop where:

  • You have tools (functions) hanging on the wall
  • You have materials (variables/data) stored on shelves
  • When you want to build something, you grab tools and materials separately
  • You have to remember which tools work with which materials

In code terms, this means:

  • Your basic building blocks are functions (like calculateArea, printResult)
  • Data (like width, length) and functions that work with that data are completely separate
  • You pass data to functions, they do their work, and give you back results

Example: Managing a Rectangle the Procedural Way

// Step 1: We declare our data separately (global variables)
// These just sit there in memory, unprotected
double width;
double length;

// Step 2: We write functions that work with this data
// But there's nothing connecting these functions to the data above!

void setWidth(double w) {
    // This function HAPPENS to work with the global width variable
    width = w;
}

void setLength(double l) {
    // This function HAPPENS to work with the global length variable
    length = l;
}

double getWidth() {
    // This function returns the global width
    return width;
}

double getLength() {
    // This function returns the global length
    return length;
}

double getArea() {
    // This function calculates area using global variables
    return width * length;
}

Using this code:

int main() {
    setWidth(5.0);      // Sets the global width variable
    setLength(3.0);     // Sets the global length variable

    cout << "Width: " << getWidth() << endl;     // Gets global width
    cout << "Length: " << getLength() << endl;   // Gets global length
    cout << "Area: " << getArea() << endl;       // Calculates area

    return 0;
}

Problems with the Procedural Approach

Problem 1: Data and Functions are Disconnected

double width;  // This variable just sits here
double length; // This variable just sits here

// These functions HAPPEN to work with those variables, but there's no formal connection!
void setWidth(double w) { width = w; }
double getArea() { return width * length; }

// Nothing stops someone from writing this by mistake:
void calculateSalary() {
    return width * length;  // OOPS! Used rectangle data in salary function!
}

Problem 2: What About Multiple Rectangles?

// If I want TWO rectangles, I need separate variables for each:
double room_width, room_length;
double garden_width, garden_length;

// And separate functions for each:
void setRoomWidth(double w) { room_width = w; }
void setGardenWidth(double w) { garden_width = w; }
void setRoomLength(double l) { room_length = l; }
void setGardenLength(double l) { garden_length = l; }

// This gets messy VERY quickly!
// What if I want 100 rectangles? 200 variables and 500 functions?

Problem 3: No Data Protection

double width = 5.0;

// Anywhere in your program, someone can do this:
width = -10.0;  // Negative width doesn't make sense!
width = 999999; // Accidentally set width to huge number

// There's no way to prevent bad data from being set

Problem 4: Difficult to Maintain

// In a large program, you might have:
double width, length, height;
double radius, diameter;
double temperature, pressure;

// And hundreds of functions scattered throughout your code
// Who knows which functions work with which variables?
// If you change one variable name, you have to find ALL functions that use it!

2. The Object-Oriented Solution: Classes üí°

What is a Class?

A class is a complex data type that groups:

  • Data (member variables/attributes)
  • Functions that work with that data (member functions/methods)

All bundled together in the same structure!

Think of a class like a blueprint for creating objects. Just like an architect creates a blueprint for houses, and then multiple houses can be built from that same blueprint.

Object-Oriented Terminology - EXPLAINED IN DETAIL

  • Class: The blueprint/template (like a cookie cutter)
    Example: The design plans for a car model (e.g., Toyota Corolla design)
  • Object: An instance of a class (like an actual cookie made from the cutter)
    Example: Your specific Toyota Corolla with license plate ABC123
  • Member Variables (Attributes): The data that defines an object's state
    Example: A car's color, engine size, current speed
  • Member Functions (Methods): The actions an object can perform
    Example: A car can start(), accelerate(), brake(), turn()

Multiple Examples Beyond Rectangle

Example 1: BankAccount Class

class BankAccount {
private:
    double balance;        // Member variable: the account balance
    string accountNumber;  // Member variable: account identifier

public:
    void deposit(double amount);   // Member function: add money
    void withdraw(double amount);  // Member function: remove money
    double getBalance();           // Member function: check balance
};

Example 2: Student Class

public:
    void setName(string n);        // Member function: set student name
    void addGrade(double grade);   // Member function: record a grade
    double calculateAverage();     // Member function: compute GPA

Example 3: Circle Class

private:
    double radius;         // Member variable: circle's radius

public:
    void setRadius(double r);      // Member function: set radius
    double getRadius();            // Member function: get radius

3. Defining a Class

Basic Class Syntax

class ClassName {    // 'class' is a C++ keyword, ClassName is what you choose
    // Class members go here
    // (both variables and functions)
};  // CRITICAL: Don't forget the semicolon! This is NOT optional!

Why the semicolon? In C++, a class definition is treated like a statement, and all statements end with semicolons. Forgetting it will cause compilation errors!

Our Rectangle Class Structure - Step by Step

Step 1: Start with the class declaration

class Rectangle {
    // Empty for now
};  // Remember the semicolon!

Step 2: Add member variables (the data)

class Rectangle {
    double width;   // This variable belongs to the class
    double length;  // This variable belongs to the class
};

Step 3: Add member function declarations

class Rectangle {
    double width;
    double length;

    // These are function DECLARATIONS (not implementations yet)
    void setWidth(double);      // Function to set width
    void setLength(double);     // Function to set length
    double getWidth();          // Function to get width
    double getLength();         // Function to get length
    double getArea();           // Function to calculate area
};

4. Access Specifiers: public and private üîê

What Are Access Specifiers?

Access specifiers are keywords that control WHO can access different parts of your class:

  • private: - Only the class's own member functions can access these members
    Think of it like: Your personal diary that only you can read
  • public: - Any part of the program can access these members
    Think of it like: A public bulletin board that anyone can read

Why Do We Need Access Specifiers?

Without access specifiers (BAD):

class Rectangle {
    double width;   // Anyone can change this!
    double length;  // Anyone can change this!
};

Rectangle r;
r.width = -50;      // Oops! Negative width!
r.length = 0;       // Oops! Zero length!

The Rectangle Class with Access Specifiers

class Rectangle {
private:    // Everything after this is PRIVATE until we say otherwise
    double width;   // ONLY Rectangle's member functions can access this
    double length;  // ONLY Rectangle's member functions can access this

public:     // Everything after this is PUBLIC until we say otherwise
    void setWidth(double);   // Anyone can CALL this function
    void setLength(double);  // Anyone can CALL this function
    double getWidth();       // Anyone can CALL this function
    double getLength();      // Anyone can CALL this function
    double getArea();        // Anyone can CALL this function
};

What Are Getter and Setter Functions?

Getter Functions (Accessors):

  • Functions that retrieve the value of a private member variable
  • They "get" the value and return it to the caller
  • Naming convention: usually start with "get" (e.g., getWidth())
  • They typically don't modify anything, just return a value

Example of a Getter:

double getWidth() {
    return width;  // Simply returns the private width variable
}

Setter Functions (Mutators):

  • Functions that modify the value of a private member variable
  • They "set" the value to whatever the caller provides
  • Naming convention: usually start with "set" (e.g., setWidth())
  • They can include validation logic to ensure data integrity

Example of a Setter with Validation:

void setWidth(double w) {
    if (w >= 0) {       // Check if width is valid
        width = w;      // Only set if valid
    } else {
        width = 0;      // Set to 0 if negative
    }
}

Why Use Getters and Setters Instead of Public Variables?

1. Data Validation:

// With setter (GOOD):
void setAge(int a) {
    if (a >= 0 && a <= 150) {  // Validate age is reasonable
        age = a;
    }
}

// Without setter (BAD):
person.age = -50;  // No validation! Invalid age!

2. Future Flexibility:

// Today: Simple getter
double getBalance() {
    return balance;
}

// Tomorrow: Add logging without changing how it's called
double getBalance() {
    logAccess("Balance checked");  // Added feature
    return balance;
}

5. The Scope Resolution Operator (::)

What is the Scope Resolution Operator?

The scope resolution operator (::) is two colons placed together. It tells C++ "this function belongs to this class."

Think of it like an address:

  • Just saying "getWidth" is like saying "123 Main Street" without the city
  • Saying "Rectangle::getWidth" is like saying "123 Main Street, Rectangle City"
  • It specifies WHICH class the function belongs to

Why Do We Need the Scope Resolution Operator?

When we define member functions OUTSIDE the class definition, we need to tell C++ which class they belong to:

// In Rectangle.h (class definition)
class Rectangle {
public:
    double getWidth();  // Just declaring the function exists
};

// In Rectangle.cpp (implementation)
double Rectangle::getWidth() {  // Rectangle:: says "this belongs to Rectangle class"
    return width;
}

Examples of Scope Resolution Operator Usage

Example 1: Simple Member Function

// Class definition
class Circle {
private:
    double radius;
public:
    double getArea();  // Declaration
};

// Implementation (outside the class)
double Circle::getArea() {  // Circle:: means "getArea belongs to Circle"
    return 3.14159 * radius * radius;
}

Example 2: Multiple Classes with Same Function Names

// Two different classes with functions of the same name
class Rectangle {
public:
    double getArea();
};

class Circle {
public:
    double getArea();
};

// Implementations - the :: tells them apart!
double Rectangle::getArea() {  // This getArea belongs to Rectangle
    return width * length;
}

double Circle::getArea() {     // This getArea belongs to Circle
    return 3.14159 * radius * radius;
}

Common Mistakes with Scope Resolution Operator

‚ùå Mistake: Forgetting the scope resolution operator

// WRONG - C++ doesn't know this belongs to Rectangle
double getWidth() {
    return width;  // ERROR: 'width' is not defined!
}

‚úÖ Correct: Including the scope resolution operator

// CORRECT - Now C++ knows this is Rectangle's getWidth
double Rectangle::getWidth() {
    return width;  // Now it can access Rectangle's width
}

6. Creating Objects (Instantiation)

What is Object Instantiation?

Instantiation means creating an actual object from a class.

Analogy:

  • Class = Recipe for chocolate cake
  • Object = An actual chocolate cake you baked using that recipe
  • Instantiation = The process of baking the cake

You can use one recipe (class) to bake many cakes (objects), and each cake is separate!

Creating Rectangle Objects

// Creating objects is just like declaring any other variable
Rectangle r1;           // Creates a Rectangle object named r1
Rectangle r2;           // Creates another Rectangle object named r2
Rectangle myRoom;       // Creates a Rectangle object named myRoom
Rectangle* ptr;         // Creates a POINTER to a Rectangle (not an object yet!)

What happens when you create an object?

  1. Memory is allocated for all the object's member variables
  2. The constructor (if any) is called to initialize the object
  3. The object is ready to use!

The Dot Operator (.)

The dot operator (.) is used to access members of an object.

Syntax: objectName.memberName

Think of it like:

  • Object = A car
  • Dot operator = "apostrophe s" in English
  • myCar.startEngine() = "my car's start engine function"

Using the Dot Operator - Examples

Rectangle room;          // Create a Rectangle object called room

// Using member functions via dot operator
room.setWidth(10.0);     // Call room's setWidth function
room.setLength(5.0);     // Call room's setLength function

// Getting values using dot operator
double w = room.getWidth();   // Call room's getWidth function
double a = room.getArea();    // Call room's getArea function

cout << "Room width: " << w << endl;
cout << "Room area: " << a << endl;

What You CANNOT Do with Private Members

Rectangle r;

// These will ALL cause compilation errors!
r.width = 10.0;         // ERROR! width is private
r.length = 5.0;         // ERROR! length is private
cout << r.width;        // ERROR! Can't access private width
double w = r.width;     // ERROR! Can't read private width

Why these errors occur:

  • width and length are marked as private
  • Private members can ONLY be accessed by the class's own member functions
  • From outside the class, you MUST use the public interface (getters/setters)

Each Object Has Its Own Data - CRITICAL CONCEPT

This is extremely important to understand:

Rectangle room;
Rectangle garden;

// Set room dimensions
room.setWidth(4.0);
room.setLength(3.0);

// Set garden dimensions
garden.setWidth(10.0);
garden.setLength(8.0);

// Each object maintains its OWN separate data!
cout << "Room area: " << room.getArea() << endl;     // Output: 12.0
cout << "Garden area: " << garden.getArea() << endl; // Output: 80.0

// Changing one doesn't affect the other
room.setWidth(20.0);
cout << "Room width: " << room.getWidth() << endl;   // Output: 20.0
cout << "Garden width: " << garden.getWidth() << endl; // Output: 10.0 (unchanged!)

Memory Visualization:

Memory:
[room object]
  - width: 20.0
  - length: 3.0

[garden object]
  - width: 10.0
  - length: 8.0

These are COMPLETELY SEPARATE in memory!

7. Constructors: Initializing Objects

The Problem with Uninitialized Objects

When we create an object like this:

Rectangle r;  // What are the values of width and length?

Without a constructor:

  • width contains garbage (whatever random data was in memory)
  • length contains garbage (whatever random data was in memory)
  • Calling r.getArea() might return 0, -472893, 8.43e+97, or crash!

What is a Constructor?

A constructor is a special member function that:

  1. Has the exact same name as the class (case-sensitive!)
  2. Has NO return type (not even void)
  3. Is called automatically when an object is created
  4. Purpose: Initialize the object's member variables to sensible values

Analogy: A constructor is like the setup process when you buy a new phone - it automatically runs to configure your device with default settings.

Types of Constructors

1. Default Constructor - Takes no parameters

class Rectangle {
private:
    double width;
    double length;

public:
    Rectangle();  // Default constructor - no parameters
};

// Implementation
Rectangle::Rectangle() {
    width = 1.0;   // Set default width
    length = 1.0;  // Set default length
}

2. Parameterized Constructor - Takes parameters to initialize

class Rectangle {
private:
    double width;
    double length;

public:
    Rectangle(double w, double l);  // Parameterized constructor
};

// Implementation
Rectangle::Rectangle(double w, double l) {
    width = w;
    length = l;
}

Constructor Initialization Lists (Advanced but Important)

There's another way to initialize member variables in constructors:

// Traditional way (assignment in body)
Rectangle::Rectangle() {
    width = 1.0;   // Assignment
    length = 1.0;  // Assignment
}

// Initialization list way (more efficient)
Rectangle::Rectangle() : width(1.0), length(1.0) {
    // Body can be empty or contain additional code
}

Why use initialization lists?

  • More efficient (initialization vs. assignment)
  • Required for const members and references
  • Clearer intent

Multiple Constructors (Constructor Overloading)

class Rectangle {
private:
    double width;
    double length;

public:
    Rectangle();                     // Default constructor
    Rectangle(double w, double l);   // Two-parameter constructor
    Rectangle(double side);          // One-parameter (square)
};

// Implementations
Rectangle::Rectangle() {
    width = 1.0;
    length = 1.0;
}

Rectangle::Rectangle(double w, double l) {
    width = w;
    length = l;
}

Rectangle::Rectangle(double side) {
    width = side;
    length = side;  // Makes a square
}

// Usage
Rectangle r1;           // Calls default constructor (1x1)
Rectangle r2(5.0, 3.0); // Calls two-parameter constructor (5x3)
Rectangle r3(4.0);      // Calls one-parameter constructor (4x4 square)

Automatic Constructor Generation

Important: If you don't write ANY constructor, C++ automatically generates a default constructor that:

  • Allocates memory for the object
  • Does NOT initialize member variables
  • Leaves primitive types (int, double, etc.) with garbage values
// Without a custom constructor
Rectangle r1;
cout << r1.getArea();  // Could output anything! (garbage * garbage)

// With a custom constructor
Rectangle r2;          // Constructor sets width=1.0, length=1.0
cout << r2.getArea();  // Output: 1.0 (predictable!)

9. Common Mistakes and How to Avoid Them ‚ùå

‚ùå Mistake 1: Forgetting the Semicolon After Class Definition

// WRONG - Missing semicolon
class Rectangle {
    // ... class members
}  // ERROR: Expected ';' after class definition

‚úÖ Correct:

class Rectangle {
    // ... class members
};  // Don't forget this semicolon!

‚ùå Mistake 2: Trying to Access Private Members Directly

Rectangle r;
r.width = 5.0;  // ERROR! width is private

‚úÖ Correct:

r.setWidth(5.0);  // Use the public interface

‚ùå Mistake 3: Calling Member Functions Without an Object

// WRONG
setWidth(5.0);  // ERROR! Which object's width?

‚úÖ Correct:

Rectangle r;
r.setWidth(5.0);  // Call setWidth on object r

‚ùå Mistake 4: Constructor Name Mismatch

class Rectangle {
public:
    rectangle();  // ERROR! Wrong capitalization
};

‚úÖ Correct:

class Rectangle {
public:
    Rectangle();  // Constructor name must match class name exactly
};

‚ùå Mistake 5: Giving Constructor a Return Type

class Circle {
public:
    void Circle();  // ERROR! Constructors have NO return type
};

‚úÖ Correct:

class Circle {
public:
    Circle();       // CORRECT! No return type at all
};

10. Practice Problems

Problem 1: Basic Class Definition

Task: Create a class called Circle with:

  • A private member variable radius
  • Public functions to set and get the radius
  • A public function to calculate the area (œÄ √ó radius¬≤)
  • A default constructor that sets radius to 1.0
class Circle {
private:
    double radius;

public:
    // Default constructor
    Circle() {
        radius = 1.0;
    }

    // Setter with validation
    void setRadius(double r) {
        if (r > 0) {
            radius = r;
        }
    }

    // Getter
    double getRadius() {
        return radius;
    }

    // Calculate area
    double getArea() {
        return 3.14159 * radius * radius;
    }
};

Problem 2: Object Creation and Usage

int main() {
